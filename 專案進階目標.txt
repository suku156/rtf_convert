📌 RTF 轉換專題 Roadmap
階段 1：核心功能完成 (MVP 已達成)

✅ 目前你已經完成：

RTF → TXT（中英文皆可讀）

RTF → WMF 圖片輸出

支援中文檔名，並建立對應資料夾存放結果

支援單檔 / 批次處理

階段 2：可靠性 & 正確性提升

🎯 目標：讓程式處理更健壯，輸出結果更穩定

 編碼處理：

自動偵測 RTF 檔案的編碼（ANSI / UTF-8 / UTF-16），避免中文亂碼。 (已經實作可分辨用的工具)

 特殊符號測試：

驗證全形符號（例：「，」「。」「★☆♥」）是否正確轉換。 (完成)

 錯誤處理：

遇到無法解析的區塊，能 跳過 + log 記錄，而不是整個程式掛掉。 

 檔案檢查：

若輸入檔案不是 RTF，要能提示錯誤並結束。

階段 3：進階輸出功能

🎯 目標：提升輸出檔案的實用性

 更多輸出格式：

.txt → .md（保留段落與結構）

.docx（用第三方庫，如 libreoffice SDK 或 docxlib）

 圖片格式轉換：

將 WMF 轉換成 PNG（用 GDI+ / Cairo / ImageMagick / FreeImage）。

 結構化輸出：

在 .txt 檔標示圖片位置，例如：

[Image1.png]
這裡是文字內容
[Image2.png]

階段 4：效率與批次處理

🎯 目標：適用於大量檔案、提升速度

 多執行緒處理：

使用 std::async 或 Thread Pool 處理多檔案，加快速度。

 進度顯示：

在處理資料夾時，顯示「已完成 X / 總數 Y」。

 日誌系統：

產生 log.txt，記錄成功 / 失敗檔案與原因。

階段 5：使用者友善度

🎯 目標：讓工具更接近「產品」

 命令列選項：

--input 指定單檔 / 資料夾

--output 指定輸出資料夾

--format txt|md|docx 選擇輸出格式

 簡易 GUI：

用 Qt / ImGui 做一個拖放式界面，讓使用者不用打命令。

 輸出結構整理：

每個 RTF 建立一個資料夾，裡面放：

text.txt
images/
  img1.png
  img2.png

階段 6：真實專案等級

🎯 目標：達到可交付 / 專業軟體標準

 完整測試樣本集：

各種 RTF 測試檔（不同版本、不同圖片、不同語系）。

 跨平台支援：

確認可在 Windows + Linux 都能執行。

 CI/CD 測試自動化（進階）：

用 GitHub Actions 自動測試，確保每次修改都不會壞掉。

 文件化：

README（功能說明、使用方式、限制）。

User Manual（圖文操作說明）。

📌 分階段完成後的成果圖景：

階段 2 → 穩定可靠，不怕特殊檔案。

階段 3 → 輸出更強大，支援 PNG/Markdown/Docx。

階段 4 → 可以處理上千檔案，速度快，還能記錄日誌。

階段 5 → 有命令列參數，甚至 GUI，使用者體驗好。

階段 6 → 真正像一個專業工具，甚至能開源或商用。
